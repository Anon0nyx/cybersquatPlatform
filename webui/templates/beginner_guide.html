First-Use Guide to Inserting & Updating Keywords

Adding Keywords

Tagging

When adding keywords it is crucial to first create an appropriate ID tag to use with the search engine to display the data collected with the keywords. There are set 3 character tags that we use and are as follows

  • med
    • For keywords associated to the Medical Sector
  • gov
    • For keywords associated to the Government Sector
  • com
    • For keywords associated to the Commercial Sector
  • pub
    • For keywords associated to the Public Sector
  • edu
    • For keywords associated to the Educational Sector
  • fin
    • For keywords associated to the Financial Sector
  • res
    • For keywords with no specific sector association (Research keywords)

Once the proper tag is chosen depending on the sector association, choose four random integers to follow the tag. A proper ID tag example would be:

med3806

Once the ID tag is created move onto determining the list of initial keywords to be used. The results produced by these keywords are critical in determining the later products of the system. It is important to take the first two iterations into account when determining the initial keywords:

First Iteration

The first iteration works by running OpenSquat with these intial keywords on the WhoISDS database of newly registered domains. So this first iteration will pull domains from a database of domains registered in the past 24hrs based on these initial keywords. Once this iteration is done the subdomains will be collected for each of these flagged domains. This is done to catch squatted domains such as 'meta[.]com' where if the only intial keyword used is 'nuharbor' this initial domain will not be flagged, so using the initial keyword 'security' along with 'nuharbor' will catch attempts such as this. It is important to take this into consideration when determining the inital keywords.

Second Iteration

Once the first iteration is finished, all of the domains that were collected have their subdomains enumerated using Sublist3r.py and then the domains and subdomains are run through OpenSquat again, but with the final keyword. Only one final keyword is permitted for later enrichment and so this keyword must be the most specific to what the aim of the collection is. For example: if looking to find domains squatted for Meta, the initial keywords may be:

  • meta
  • facebook
  • (And any additional broad keywords)

And the final keyword could be just simply 'meta'. This set of keywords would allow for subdomains such as the one aforementioned that would not be flagged by only using the initial keyword 'meta' but would be flagged with the initial keywords 'meta'

Updating Keywords

Updating keywords is simple if the ID tag for those keywords is known, which is why it is critical to remember the ID tag used, as it only requires entering the new intial keywords and final keyword along with the ID tag and the API updates the database accordingly. This would be done in instances where perhaps an initial keyword is forgotten, or if one is thought-up later on.
